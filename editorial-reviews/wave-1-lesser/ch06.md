## Syntax trees

### Overview
The colloquial register mostly works here because it is tied to concrete examples and an argument that knows where it is going. But the chapter is padded with repeated scene-setting and procedural restatement, so the prose too often explains what it has already explained. One copy-paste slip in the top-down example is worse than a stylistic lapse: it briefly breaks the reader's trust.

### Flags
1. **Line ~35** [OVERWRITING]: "This chapter guides you through the basic principles of syntax trees. It will show you how to read and construct these diagrams..." -- The learning objectives have already done this work. You do not need another soft-focus announcement of the chapter's intentions.
2. **Line ~37** [THROAT-CLEARING]: "By the end of this chapter, you'll have a new perspective on sentence structure..." -- This is generic textbook uplift. It delays the start instead of sharpening it.
3. **Line ~127** [TONAL SLIP]: "With trees, it is obvious when the analysis is correct." -- Too absolute. Trees can clarify an argument; they do not make syntactic judgment self-evident.
4. **Line ~208** [DEAD]: "This VP has a head:V, and terminates in helped. There's also you..." -- This is a stray sentence from another example. In a walkthrough of *I had my breakfast*, it makes the prose momentarily unusable.
5. **Line ~216** [DEAD]: "Finally, we'll add the object:NP, with its head:N... and its terminal you." -- Same problem. Once the example has changed, these instructions are not merely wrong but distracting.
6. **Line ~237** [OVERWRITING]: "As always, the top node is a phrasal category..." -- This refrain becomes padding. By this point the reader does not need the same opening sentence for each miniature tree.
7. **Line ~286** [TONAL SLIP]: "As always -- and by now you can probably finish this sentence for me -- ..." -- The aside knows the prose is repetitive, but instead of cutting the repetition it turns it into a wink. That is a slightly false chumminess.
8. **Line ~323** [UNDERWRITING]: "To identify constituents, you can use several constituency tests." -- The section needs one plain sentence saying that these are heuristics, not infallible diagnostics, and that they do not always converge. Without that, the brisk list overpromises.
9. **Line ~497** [OVERWRITING]: "This chapter introduced syntax trees as essential tools for understanding and teaching English grammar structure:" -- The summary re-expands the chapter instead of compressing it. After so much guided repetition above, this feels like one lap too many.
